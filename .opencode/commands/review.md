# مراجعة الكود

## ماذا يفعل؟
1. يراجع جميع التغييرات غير الم committed (`git diff`)
2. يفحص الأخطاء المنطقية والأمنية
3. يتأكد من نمط الكود والاختبارات
4. يقدم تقريراً كاملاً

## قائمة الفحص
- [ ] لا أخطاء واضحة (syntax, logic)
- [ ] اتباع نمط الكود (PEP 8, ESLint...)
- [ ] اختبارات كافية (unit tests)
- [ ] لا ثغرات أمنية (API keys, SQL injection)
- [ ] type hints صحيحة
- [ ] إدارة الأخطاء (try/except)
- [ ] لا تعليقات مهملة أو dead code

## خطوات التنفيذ
1. استخدم `git diff` لرؤية التغييرات
2. استخدم مهارة `code-review` للفحص المتقدم
3. افحص الأمان (API keys في الكود؟)
4. افحص الاختبارات (هل فيه اختبارات للتغيير الجديد؟)
5. اعرض تقريراً للمستخدم مع نقاط القوة والضعف
